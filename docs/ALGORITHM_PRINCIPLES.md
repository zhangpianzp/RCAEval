# RCAEval 算法原理说明

本文档用于解释 RCAEval 中主要方法的核心思路、输入依赖和适用场景，帮助你在复现实验或扩展方法时更快选型。

说明范围：以当前仓库实现为准。

## 1. 统一问题定义与输出

RCAEval 大多数方法都遵循相同任务形式：

1. 输入一段包含故障前后窗口的时序数据（常见列为服务指标，另含 time）。
2. 根据注入时间或异常区间划分 normal/anomalous。
3. 对候选根因实体（服务指标、操作、节点）进行打分。
4. 返回排序结果 ranks（从最可疑到最不可疑）。

常见返回结构：

- ranks: 根因候选列表
- node_names: 节点名称列表（部分方法提供）
- adj: 学到的图结构（部分图方法提供）

## 2. 通用处理流水线

多数方法会复用时间序列预处理工具（见 RCAEval/io/time_series.py），核心步骤通常包括：

1. 清洗时间列、常数列和冗余列。
2. 对 normal 与 anomalous 使用同一套特征集合（取列交集）。
3. 对缺失值/异常值做填充与裁剪。
4. 进入打分阶段（统计、图推断或深度模型）。

这也是你开发新方法时建议遵循的输入规范。

## 3. 方法家族与核心原理

## 3.1 统计偏差类（快、解释直观）

代表方法：baro、nsigma、mmbaro、mmnsigma

核心思想：

1. 以正常窗口作为参考分布。
2. 计算异常窗口偏离程度并排序。

典型实现信号：

- z-score 或稳健标准化后的极值分数
- n-sigma 异常点计数
- 多源数据下的加权融合分数

优点：

- 实现简单、速度快。
- 适合先做粗排查。

局限：

- 对传播链路和间接因果建模较弱。

## 3.2 因果图 + 节点排序类（结构化、可分析传播）

代表方法：pc_pagerank、fci_pagerank、ges_pagerank、granger_pagerank、lingam_pagerank、pc_randomwalk 等

核心思想：

1. 先通过图构建算法学习变量关系图（PC/FCI/GES/Granger/LiNGAM/NOTEARS 等）。
2. 再在图上运行排序头（PageRank 或 RandomWalk）得到可疑度。

常见排序形式：

- PageRank：迭代传播节点重要度。
- RandomWalk：统计从异常信号出发的到达频率。

优点：

- 能体现“异常传播路径”。
- 便于结合图可视化做分析。

局限：

- 图学习对样本量、噪声和超参数较敏感。
- 计算成本通常高于纯统计法。

## 3.3 局部因果发现类（大规模更实用）

代表方法：rcd、mmrcd、easyrca、circa

核心思想：

1. 引入故障指示变量（常称 F-node）或异常窗口标签。
2. 优先寻找与故障变量直接相关的局部邻域，而非全图。
3. 在局部结构中给出根因候选排序。

优点：

- 比全图发现更可扩展。
- 对线上场景更友好。

局限：

- 依赖局部假设成立。
- 参数（如 alpha）对结果影响明显。

## 3.4 深度学习重构/表示学习类（表达力强）

代表方法：mscred、causalrca

核心思想：

1. 通过神经网络学习时空结构或隐变量关系。
2. 使用重构误差或学得图结构进行根因排序。

优点：

- 对复杂模式的表示能力较强。

局限：

- 训练成本高、超参数多。
- 可解释性通常弱于显式图方法。

## 3.5 Trace/调用链专用类（链路信息充分时效果好）

代表方法：microrank、tracerca

核心思想：

1. 从 span 或请求-操作关系中构建结构化图。
2. 使用二部图排序、SLO 违反统计等策略打分。

优点：

- 直接利用调用链拓扑，定位服务传播链路更自然。

局限：

- 对 trace 完整性和质量依赖较高。

## 5. 不同算法的权重如何合理分布

这一节回答“权重怎么设才合理”。先给结论：权重分配应与方法机制一致，而不是统一套一个比例。

### 5.1 当前实现中的典型权重机制

1. 统计偏差类（baro/nsigma/mmbaro/mmnsigma）
- 显式权重：通常没有单独的可学习权重。
- 实际机制：每个特征列先做标准化后比较异常分数，再统一排序；多源版本本质是把多源列拼接后共同参与排序。
- 启示：如果不做源内归一化，特征数量更多的数据源会在排序中占更大影响。

2. PageRank 类（pc/fci/ges/granger/lingam + pagerank）
- 显式权重：阻尼系数 damping_factor（默认 0.85）。
- 实际机制：
	- $0.85$ 表示更多依赖图结构传播。
	- $0.15$ 表示保留随机跳转，防止被局部环路过度放大。
- 启示：图结构可靠时可保持高阻尼；图噪声高时可适当降低。

3. RandomWalk 类
- 显式权重：$\rho$（父子边转移权重，默认 0.5）、二阶游走中的 $\beta$（历史路径权重，默认 0.5）。
- 实际机制：
	- $\rho$ 越高，越偏向沿子节点传播。
	- $\beta$ 越高，越强调当前转移概率；越低越强调上一步记忆。

4. TraceRCA
- 显式权重：support 与 confidence 用对称调和形式融合。
- 融合公式：$JI = \frac{2sc}{s+c}$。
- 启示：该公式天然对两者等权，只有同时高才会得到高分。

5. MicroRank
- 显式权重：$d=0.85$（传播强度）与 $\alpha=0.01$（服务间传播项系数）。
- 实际机制：请求-操作二部图中同时考虑结构传播与请求侧先验。

### 5.2 推荐的“合理分布”原则

1. 先统一尺度，再谈权重
- 若不同数据源量纲不同，先做源内标准化（z-score、robust scaling），再做源间融合。

2. 按“信号质量”分配，不按“个人偏好”分配
- 可用质量指标：缺失率、噪声率、时钟对齐度、异常覆盖率。
- 建议将源权重设置为质量分数的归一化：
	$$w_i = \frac{q_i}{\sum_j q_j}$$

3. 保持可解释的稀疏权重
- 对不稳定或缺失严重的数据源，给低权重而不是硬删除，避免完全丢失补充证据。

4. 用验证集做权重选择
- 在历史故障集上做网格搜索或贝叶斯优化，目标可设为 Avg@5 / MRR。

### 5.3 多源场景的实用起始配比

在没有历史先验时，可用以下“冷启动”配比作为初值（后续再调优）：

1. 指标主导场景：metrics/logs/traces = 0.6/0.25/0.15
2. 链路主导场景：metrics/logs/traces = 0.4/0.2/0.4
3. 日志噪声较大场景：metrics/logs/traces = 0.55/0.15/0.30

注意：这些是融合层权重建议，不替代各方法内部的固定参数（如 PageRank 的 damping）。

### 5.4 什么时候该调整权重

出现以下现象时，建议重新分配权重：

1. Top-K 结果长期被单一数据源“垄断”。
2. 切换数据集后性能波动明显（说明权重过拟合某一系统）。
3. 引入新数据源后整体指标下降（说明新源噪声大于信息增益）。

## 6. 选型建议

如果你的目标是：

1. 快速 baseline 和高吞吐评估：优先 baro/nsigma。
2. 解释传播链路：优先图方法（pc/fci/ges/granger/lingam + pagerank/randomwalk）。
3. 变量很多且需控制开销：优先 rcd/mmrcd/easyrca 这类局部方法。
4. trace 质量高：优先 microrank/tracerca。
5. 有训练资源并希望捕获复杂模式：尝试 mscred/causalrca。
